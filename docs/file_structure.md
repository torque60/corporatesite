# corporatesite ディレクトリ構成マップ

このドキュメントでは、`corporatesite` ディレクトリの構造とその役割について説明します。本プロジェクトは、機能ごとにディレクトリを分割し、管理のしやすさを重視して構成されています。

## 1. フォルダ構成の概要

プロジェクトの主要な構成は以下の通りです。

```text
corporatesite/
├── index.html          # トップページ
├── about/              # 会社概要セクション
├── assets/
│   └── img/         # 共通画像リソース
├── contact/            # お問い合わせセクション
├── css/                # スタイルシート格納
├── js/                 # JavaScriptファイル格納
├── locations/          # 拠点・アクセス情報セクション
├── products/           # 販売品目セクション
│   ├── detail/         # 製品個別詳細
│   │   ├── catalog/    # 製品カタログ
│   │   └── img/        # 製品別イメージ
│   └── other-detail/   # その他販売品目
│
└── docs/               # プロジェクトドキュメント
    ├── README.md       # ドキュメント全体の概要
    ├── file_structure.md # 構成図
    ├── git_commands.md # Gitコマンド一覧 (Markdown版)
    └── git_commands_plain.txt # Gitコマンド一覧 (テキスト版)
```

---

## 2. 各要素の役割

### 2.1. ルート要素
- **index.html**: サイトのメインエントランスです。各セクションへのナビゲーションと、最新情報や事業の全体像を提示します。

### 2.2. コンテンツディレクトリ
- **about/**: 会社概要、理念、代表挨拶などの企業情報を管理するディレクトリです。
- **contact/**: ユーザーからの問い合わせ窓口となるページを管理します。
- **locations/**: 事業所一覧や地図情報など、物理的な拠点情報を集約しています。
- **products/**: 製品情報の核となるディレクトリです。製品個別詳細（`detail/`）や、補足的な製品情報（`other-detail/`）などのサブディレクトリで構成されています。

### 2.3. システム・リソースディレクトリ
- **css/**: サイト全体のデザインを定義するスタイルシートを格納します。BEM設計に基づいた共通コンポーネントの定義が含まれます。
- **js/**: インタラクティブな動作やアニメーションを制御するスクリプトを格納します。
- **assets/img/**: UIパーツや共通して使用される画像アセットを管理します。

### 2.4. ドキュメントディレクトリ
- **docs/**: プロジェクトの運用や構成に関する各種ドキュメントを管理します。開発環境のセットアップ手順やGitの使い方のルールなどを集約し、チーム内の知識共有を円滑にします。
